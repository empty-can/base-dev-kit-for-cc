# 調査F: Hooks のクロスプラットフォーム実装可能性

## 調査概要

**調査目的**: Hooks をベースキットに含めるかの最終判断のため、クロスプラットフォーム実装可能性を検証
**実施日**: 2026-04-24

## 主要な発見

### 1. 仕様確認
- hook の `command` は shell コマンドとして実行
- デフォルト shell: bash（`"shell": "powershell"` で PowerShell 指定可能）
- 環境変数サポート: `$CLAUDE_PROJECT_DIR`、`${CLAUDE_PLUGIN_ROOT}` 等
- exit code 0: 成功、exit code 2: ブロック（stderr が Claude へ返される）

### 2. 公式実装パターン
**claude-code-action の例**:
```json
{
  "hooks": {
    "PostToolUse": [
      {
        "hooks": [{"type": "command", "command": "bun run format"}],
        "matcher": "Edit|Write|MultiEdit"
      }
    ]
  }
}
```
→ `bun run format` は OS 中立（Node.js ecosystem）、単一定義で全 OS 対応

### 3. クロスプラットフォーム実現案の評価

| 案 | 実現可能性 | 備考 |
|---|---|---|
| 案1: OS 中立コマンドのみ（git/echo 等） | ✅ 完全に実現可能 | 公式パターンと一致 |
| 案2: OS 判定分岐（Bash + uname） | ⚠️ Windows で不確実 | WSL/Git Bash 前提 |
| 案3: Node.js スクリプト bundled | ✅ 実現可能 | node 依存が追加される |

## 結論

**✅ Hooks をベースキットに含めるべき（条件付き）**

**条件**:
- 案1（OS 中立コマンドのみ）で実装
- `git`, `echo` など Windows/Mac/Linux 全てで動作するコマンドに限定
- 複雑な bash スクリプトは避ける
- シンプルな demonstrate 用サンプル 1 つに留める

**推奨サンプル** (最小限・OS 中立):
```json
{
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "git status --short"
          }
        ]
      }
    ]
  }
}
```

**根拠**:
- 公式の claude-code-action リポジトリで実装実績あり
- Hooks の学習価値が高い（セキュリティ用途への拡張性）
- OS 中立コマンドだけでも十分に「動作する実用サンプル」になる
