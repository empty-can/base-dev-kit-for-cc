# 調査B: .claude/ サブフォルダ構造のベストプラクティス

## 調査概要

**調査目的**: Claude Code の `.claude/` ディレクトリで公式にサポートされるサブフォルダ構造を把握し、ベースキット設計の指針とする

**実施日**: 2026-04-24

## 公式サポートされるサブフォルダ一覧

| サブフォルダ | 用途 | 認識条件 | ファイル形式 |
|---|---|---|---|
| `CLAUDE.md` | プロジェクト固有の指示 | `.claude/CLAUDE.md` または `./CLAUDE.md` | Markdown |
| `settings.json` | パーミッション・フック・環境変数・モデル設定 | `.claude/settings.json` | JSON |
| `settings.local.json` | 個人用プロジェクト設定（gitignore対象） | `.claude/settings.local.json` | JSON |
| `rules/` | トピック別条件付き指示 | `*.md` ファイルを自動発見 | Markdown（YAML frontmatter） |
| `skills/` | 再利用可能なプロンプト | `<name>/SKILL.md` で自動発見 | Markdown（YAML frontmatter） |
| `commands/` | 単一ファイルのコマンド（legacy） | `*.md` ファイルを自動発見 | Markdown |
| `agents/` | 専門化されたサブエージェント定義 | `*.md` ファイルを自動発見 | Markdown（YAML frontmatter） |
| `agent-memory/` | サブエージェントの専用永続メモリ | `<agent-name>/MEMORY.md` | Markdown |
| `output-styles/` | カスタム出力スタイル | `*.md` ファイルを自動発見 | Markdown（YAML frontmatter） |

## 各サブフォルダの要点

### CLAUDE.md
- 配置場所: `./CLAUDE.md` または `./.claude/CLAUDE.md`（等価）
- **推奨**: プロジェクトルート `./CLAUDE.md`（混乱回避）
- 200 行以内を target、長すぎると `rules/` に分割

### rules/
- トピック別・パス別の条件付き指示
- YAML frontmatter の `paths:` で glob パターン指定可能
- パス未指定時は CLAUDE.md と同様にセッション開始時に読み込み
- サブディレクトリ対応（例: `rules/frontend/react.md`）

### skills/
- `skills/<name>/SKILL.md` 構造
- 支援ファイル（templates, scripts, examples）をバンドル可
- 動的監視（ファイル追加/削除が即座に反映）
- Custom commands は skills に統合された（`commands/deploy.md` と `skills/deploy/SKILL.md` は等価。skill が優先）

### commands/
- 単一ファイルの Slash Commands
- レガシー位置づけ（新規は skills/ 推奨）
- 同名の skill と command 両方あれば skill が優先

### agents/
- サブエージェント定義
- 独立した context window で実行
- ツール制限・専用モデル指定可
- `memory: project|local|user` で `agent-memory/` 配下に専用メモリ作成

### output-styles/
- Claude の応答方式をカスタマイズ
- `/config` から選択可能
- セッション開始時反映（mid-session 変更は効かない）

## 推奨ディレクトリレイアウト

### 最小構成
```
claude-code-base-kit/
├── CLAUDE.md
├── .gitignore
└── .claude/
    └── settings.json
```

### 推奨構成（ベースキット向け）
```
claude-code-base-kit/
├── CLAUDE.md
├── .gitignore
└── .claude/
    ├── settings.json
    ├── settings.local.json   (gitignore)
    ├── rules/
    │   └── coding-standards.md
    ├── skills/
    │   └── <sample-skill>/
    │       └── SKILL.md
    └── agents/
        ├── code-reviewer.md
        └── orchestrator.md
```

### フル機能構成
上記 + `commands/` / `agent-memory/` / `output-styles/` / `settings.json` の `hooks` プロパティ（単独フォルダは存在しない）

## 命名規則

| 対象 | 形式 | 例 |
|---|---|---|
| CLAUDE.md | 固定 | `CLAUDE.md` |
| rules | kebab-case.md | `code-style.md`, `api-design.md` |
| skills | ディレクトリ名 kebab-case + `SKILL.md` | `deploy/SKILL.md`, `security-review/SKILL.md` |
| commands | kebab-case.md | `fix-issue.md`, `release.md` |
| agents | kebab-case.md | `code-reviewer.md`, `orchestrator.md` |
| output-styles | kebab-case.md | `teaching-mode.md` |
| settings.json | 固定 | `settings.json` |
| settings.local.json | 固定 | `settings.local.json` |

## 避けるべきパターン

1. **同名の skill と command の併用** — skill が優先される。commands/ は legacy
2. **CLAUDE.md の重複配置** — project root のみ推奨
3. **rules/ で `paths:` 指定なし** — 全 rule がセッション開始時ロードされ無駄
4. **plugin 内構造の誤り** — `.claude-plugin/` 内に skills/agents を置かない
5. **支援ファイルを SKILL.md に埋め込み** — 別ファイル化して遅延ロード

## ベースキットへの推奨事項

1. **CLAUDE.md はプロジェクトルート `./CLAUDE.md`** に配置（`.claude/CLAUDE.md` は使わない）
2. **skills/ を使う（commands/ は避ける）** — 将来性のため
3. **rules/ で path-specific rules** をサンプルとして提供
4. **agents/ に orchestrator と code-reviewer** のサンプルを用意
5. **settings.local.json は空または最小限** のサンプルを用意（gitignore に登録）
6. **agent-memory/ はデフォルトで作らない** — 必要に応じて agent が自動生成
7. **output-styles/ は今回除外** — オプション機能のため
