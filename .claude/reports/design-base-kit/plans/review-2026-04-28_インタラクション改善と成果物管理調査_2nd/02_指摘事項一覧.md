## 凡例

- **CRITICAL**: 内容が誤っており、ベースキット設計判断や読者の理解を誤らせる可能性が高い
- **IMPORTANT**: 裏取り不足・整合性ギャップ・追跡性不足のため放置すると後続作業の根拠が弱くなる
- **SUGGESTION**: 改善余地。プロセス・テンプレート改善の提案
- **POSITIVE**: 強みとして残す観点（修正不要だが今後も継承すべき）
- **`[未確認]`**: 一次資料による直接確認ができず、推測・既存知識ベースで記述した指摘内容に付与

### C2-01: B 表 59 行目 + 末尾 344 行目 — 画像 Ctrl+V ペーストの `[未確認]` 判定が誤り、Windows デフォルトキーも事実誤認の可能性

- **対象**:
  - 59 行目「画像・スクリーンショットを渡す: ... `Win+Shift+S` → `Ctrl+V` ... 参考: `cloudnative-co/claude-code-starter-kit` README `[未確認]`: 公式ドキュメントへの明示的記載なし」
  - 344 行目「Ctrl+V によるスクリーンショット/画像ペーストのサポート（`cloudnative-co` README に記載あり。公式 Claude Code ドキュメントへの明示的記載は未確認）」
- **内容**: 公式 `keybindings.md` の **Chat actions 表に明示記載がある**：
  ```
  chat:imagePaste | Ctrl+V (Alt+V on Windows) | Paste image
  ```
  したがって以下 2 点が事実と異なる：
  1. 「公式ドキュメントへの明示的記載なし」は誤り。`keybindings.md#chat-actions` に存在
  2. **Windows のデフォルトキーは `Alt+V`** であり、`Ctrl+V` ではない。`Ctrl+V` を Windows で押すと、ターミナルの貼り付け処理に取られて Claude Code 側で画像ペーストとして認識されない可能性が高い
- **検証根拠**: WebFetch `https://code.claude.com/docs/en/keybindings` の Chat actions 表に上記アクションを直接確認。同ページ「Chat actions」セクションは `chat:submit` `chat:newline` 等と並列で `chat:imagePaste` を定義
- **対応案**:
  1. 59 行目の操作列を「macOS: `Cmd+Shift+Ctrl+4` → `Ctrl+V` / Windows: `Win+Shift+S` → **`Alt+V`**」に訂正
  2. 59 行目の出典列を `[keybindings.md: "Chat actions"](https://code.claude.com/docs/en/keybindings#chat-actions)` に変更し、`[未確認]` タグを削除
  3. 344 行目の項目を削除（または「Windows でのデフォルトは `Alt+V`」事実を別形態で残すなら、別の確認済みノートとして書き換え）
- **優先度**: 高（コピペで動かない記述を提示している恐れ）

### C2-02: Safety Net 補足 280 行目 — matcher `tool == "Bash"` は公式 `hooks.md` 仕様外の式形式

- **対象**: 280 行目
  ```json
  "PreToolUse": [{
    "matcher": "tool == \"Bash\"",
    "hooks": [{"type": "command", "command": "cc-safety-net --claude-code"}]
  }]
  ```
- **内容**: 公式 `hooks.md` の matcher 仕様は以下のみを許容：
  - 単純なツール名（`"Bash"`, `"Edit"`）
  - パイプ区切り（`"Edit|Write"`）
  - 正規表現（`"^Notebook"`）
  - MCP ツールパターン（`"mcp__memory__.*"`）
  - 全マッチ（`""`, `"*"`, または省略）
  
  `tool == "Bash"` のような式形式は **公式仕様にない**。これは cloudnative-co の `features/safety-net/hooks.json` 8 行目から逐語コピーされており、cc-safety-net パッケージ内部で独自にパースしている可能性は否定できないが、Claude Code 本体は `"tool == \"Bash\""` という文字列をツール名として解釈し、どの PreToolUse 呼び出しにもマッチしない可能性が高い
- **検証根拠**:
  - WebFetch `https://code.claude.com/docs/en/hooks` の matcher 仕様確認（「ツール名指定のみが公式に許容される。式形式は許容されない」）
  - 一次資料 `C:\workspace\claude-doc-repositories\cloudnative-co\claude-code-starter-kit\features\safety-net\hooks.json` 直接読み込み（同じ matcher 文字列を確認）
  - cc-safety-net は外部 npm パッケージ（`npm install -g cc-safety-net`）であり、Claude Code 公式のフック仕様の一部ではない
- **対応案**:
  - 公式仕様準拠の `"matcher": "Bash"` に差し替える（cc-safety-net コマンド側で git/rm のサブコマンド判定はおこなうため、フック側は Bash 全マッチで問題ない）
  - もしくは現行表記を維持する場合は「※ `tool == "Bash"` 形式は cc-safety-net 独自の matcher 表記。公式 hooks 仕様では `"Bash"` と単純なツール名指定が標準。実際の動作は cc-safety-net 実装に依存」と明示注記し、`[未確認]` タグを付与
- **優先度**: 高（読者がコピペして実装した際、フックが発火しない可能性）

- **対象**: 122 行目「セッション途中で名前を変更 / `/rename [feature-name: current-status]` / 出典: [cli-reference.md](https://code.claude.com/docs/en/cli-reference#cli-flags)」

- **対応案**:
  - 引用先を `[cli-reference.md: --name/-n](https://code.claude.com/docs/en/cli-reference#cli-flags)` に修正（`--name` フラグ説明内で `/rename` が言及されている事実は正しい）
  - もしくは `slash-commands.md` 等のスラッシュコマンド一次資料を別途参照
- **優先度**: 中（再検証コスト増・引用節の正確性）
