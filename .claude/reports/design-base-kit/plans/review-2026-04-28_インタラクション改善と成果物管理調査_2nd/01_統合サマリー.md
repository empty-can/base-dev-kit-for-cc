### 弱み（今回新たに発見されたもの）

1. **画像 Ctrl+V ペーストの `[未確認]` 判定が誤り（CRITICAL）**
   - 59 行目および 344 行目で「公式ドキュメントへの明示的記載なし」と記述されているが、公式 `keybindings.md` の Chat actions 表に `chat:imagePaste | Ctrl+V (Alt+V on Windows) | Paste image` として**明示記載されている**。
   - さらに **Windows のデフォルトキーは Alt+V**（`Ctrl+V` ではない）。報告書の `Windows: Win+Shift+S → Ctrl+V`（59 行目）は事実誤りの可能性が高い。

2. **Safety Net 設定例の matcher 形式が公式仕様外（CRITICAL）**
   - 280 行目の `"matcher": "tool == \"Bash\""` は公式 `hooks.md` の仕様にない式形式。公式に許容される matcher は「ツール名そのもの」（`Bash`）、パイプ区切り（`Edit|Write`）、正規表現、MCP パターン（`mcp__memory__.*`）のみ。
   - これは cc-safety-net パッケージ内部での独自記法か、cloudnative-co の hooks.json 由来の記法（`features/safety-net/hooks.json` の 8 行目で同じ式形式を使用）。`tool == "Bash"` を Claude Code が解釈するわけではない。
   - 公式仕様だと誤解した読者がコピペすると **動作しない**。

3. **`/rename` の引用先誤り（IMPORTANT）**
   - 122 行目の `/rename` の出典 `[cli-reference.md](https://code.claude.com/docs/en/cli-reference#cli-flags)` は CLI フラグのドキュメント。`/rename` はインタラクティブセッション内のスラッシュコマンドであり CLI フラグではない。`cli-reference.md` 内では `--name/-n` フラグ説明欄で「`/rename` changes the name mid-session」と言及されているのみ。引用アンカーとしては `#cli-commands` か、より望ましくは `slash-commands.md` 等のスラッシュコマンド一次資料を参照すべき。

4. **「未確認項目」セクションと本文の整合性ギャップ（IMPORTANT）**
   - 344 行目「Ctrl+V によるスクリーンショット/画像ペーストのサポート（公式 Claude Code ドキュメントへの明示的記載は未確認）」は事実誤認。CRITICAL 1 と連動して解消が必要。
