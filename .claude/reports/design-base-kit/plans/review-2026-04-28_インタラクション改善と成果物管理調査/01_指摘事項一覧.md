### I-08: 中間成果物 ① 132〜144 行目 — `git checkout --orphan` 後のクリーンアップ手順抜け

- **対象**: 132〜144 行目「孤立ブランチ（orphan）を remote に push」のサンプルコード
- **内容**: `git-checkout` の `--orphan` 仕様（WebFetch 結果）：
  > "If you want to start a disconnected history that records a set of paths that is totally different from the one of <start-point>, then you should clear the index and the working tree right after creating the orphan branch by running `git rm -rf .` from the top level of the working tree."
  
  報告書のサンプルは：
  ```
  git checkout --orphan work-notes/feature-name
  git add .claude/reports/
  git commit -m "wip: ..."
  ```
  
  この順序では、orphan 直後に index には `<start-point>`（main 等）の内容が staged のまま残るため、`.claude/reports/` 以外のファイルもコミットされる可能性がある。
- **対応案**: サンプルを次のように修正：
  ```
  git checkout --orphan work-notes/feature-name
  git rm -rf --cached .          # ← 追加: index から既存ファイルを除外
  git add .claude/reports/
  git commit -m "wip: ..."
  git push origin work-notes/feature-name
  ```
- **優先度**: 中
