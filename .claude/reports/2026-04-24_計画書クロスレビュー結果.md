# 計画書クロスレビュー結果

## 実施概要

**実施日**: 2026-04-24
**レビュアー**: Plan エージェント（Opus 4.7 別インスタンス、独立コンテキスト）
**対象**: `.claude/plans/base-kit-構成プラン.md`（v2）
**判定**: ⚠️ 条件付き承認

## 重大な問題

### 1. Orchestrator エージェントの実現可能性が未検証
- Sub-agents から別 Sub-agents を呼び出す公式 API が計画書で未裏取り
- 調査D でも先行事例ゼロ
- **対応**: 追加調査で公式仕様を確認してから実装方針決定（対応案 A=Skill 化、B=追加調査）

### 2. 秘匿情報管理サンプルの欠落
- 確定要件に「環境変数・秘匿情報管理」があるのに `.env.example` と `CLAUDE.local.md.example` が不在
- `settings.local.json.example` のみでは片手落ち
- **対応**: タスク 9 として追加

### 3. Hooks 不採用の論拠が弱い
- 33% 採用率は不採用根拠にならない（Skills も 33% だが採用している）
- hooks は bash 依存ではなく、クロスプラットフォームで書ける可能性あり
- 引き継ぎ文書では「保留項目」だったのにユーザー再確認なしに確定している
- **対応**: 追加調査でクロスプラットフォーム実装可能性を確認後に最終判断

## 改善提案

1. `rules/coding-standards.md` の `paths: "**/*"` は無意味 → 言語拡張子指定に変更
2. `gh pr create` のクロスプラットフォーム懸念（Windows で未プリインストール）
3. `WebFetch(domain:github.com)` は広すぎる → `anthropics/*` 等に絞る
4. Agent Teams の調査が浅い
5. 検証方法の `node -e` が Node.js 依存
6. `.gitignore` の `agent-memory-local/` の根拠不明
7. `.gitignore` の言語別エントリが「言語中立」原則と不整合
8. 検証方法に実動作確認が不足

## 良かった点

4. `.example` 方式
5. 後回し論点セクションの存在
