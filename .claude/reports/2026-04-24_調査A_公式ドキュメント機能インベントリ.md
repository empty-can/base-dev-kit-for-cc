## 主要所見サマリ（ベースキット設計に直結）

### 必須機能
1. **CLAUDE.md** — `./CLAUDE.md` または `./.claude/CLAUDE.md` に配置。200 行以内 target。
2. **settings.json（4 層スコープ）** — Managed > CLI > Local > Project > User の優先度
3. **権限管理（permissions）** — allow / deny / ask ルール。`Tool(specifier)` パターンマッチ
4. **セッション管理** — `--continue` / `--resume` で再開可能

### 推奨機能
5. **Hooks** — 14 種類のイベント（SessionStart / PreToolUse / PostToolUse / Stop / UserPromptSubmit など）
6. **Skills** — `.claude/skills/<name>/SKILL.md`。オンデマンド読み込み（context 効率）
7. **Sub-agents** — `.claude/agents/<name>.md`。独立コンテキスト・専用権限
8. **MCP** — 外部ツール連携。`defer_tool_definitions: true` でコンテキスト節約
9. **Sandbox** — filesystem + network 隔離。approval fatigue 対策

### 重要な最新情報
- **Custom commands は skills に統合された**: `.claude/commands/X.md` と `.claude/skills/X/SKILL.md` は等価
- **Auto Memory**: `~/.claude/projects/<project>/memory/` に自動蓄積（200 行上限）
- **Path-specific rules**: `.claude/rules/*.md` に YAML frontmatter `paths:` で範囲指定
- **Plan Mode**: 実装前の計画レビュー機能（Shift+Tab で切替）

## ベースキット設計への含意

- **最小必須セット**: CLAUDE.md + settings.json + .gitignore
- **推奨セット**: 上記 + hooks + skills + agents + MCP
- **「動作する実用サンプル」を作るなら**: skills と agents の両方でサンプルを用意すべき
- **言語中立にするなら**: コマンド allow list を最小限に、skills/agents の内容を言語に依存させない
